**Intent:** Create. New PRD for PokerMath, referencing the existing final product brief at `_bmad-output/planning-artifacts/briefs/brief-PokerMath-2026-05-28/brief.md`.

**Inherited context from brief (load-bearing):**
- Primary audience: math-oriented learner using poker as a transfer-of-learning context (NOT a poker player wanting to improve). Poker player is an acknowledged secondary/future profile.
- Bloom's revised taxonomy is the course-prescribed structural foundation, not a design choice.
- 15–30 minute single-session completion is a course requirement.
- Three sequential learning objectives: (LO1) equity via Rule of 2-and-4, (LO2) pot odds, (LO3) call profitability judgment.
- Wrong-answer behavior: context-aware canned hint, retry allowed, correct answer not revealed.
- Platform: desktop-only web, no responsive/mobile.
- Session model: single session, no save/resume, no accounts, no scoring persistence.
- Out of scope: drill mode / procedural generation, pre-flop & multi-street strategy, branching/adaptive paths, scoring/persistence/accounts.

**Design spec PDF reviewed** (`HCI 520 Term Project Design Specifications-2.pdf`). Extractions:

- Wireframe — assessment screen uses **multiple labeled input fields** (e.g. # outs, # remaining streets, hand equity) + single "Check Answer" button; board & hand rendered as **card graphics** with suit symbols/colors. Back / Next buttons for linear nav.
